## 1. Context & Purpose
This design specifies the bootstrap foundation of the Horizon repository, establishing the package skeletons, shared domain/security abstractions, Flyway schema migrations, frontend testing setup, and setup documentation. 

No business functionality is introduced in this phase.

## 3. Database Schema & Flyway Migration
We will create a Flyway schema migration script:
`server/src/main/resources/db/migration/V20260620200000__bootstrap_schema.sql`

It contains the schema definition for:
1. `users` table: including `first_name` and `last_name` columns.
2. Spring Session tables: standard PostgreSQL definitions for session clustering.

```sql
-- 1. Users Table
CREATE TABLE users (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY,
    email VARCHAR(255) NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    first_name VARCHAR(255) NOT NULL,
    last_name VARCHAR(255) NOT NULL,
    created_at TIMESTAMPTZ NOT NULL,
    updated_at TIMESTAMPTZ NOT NULL,
    CONSTRAINT pk_users PRIMARY KEY (id),
    CONSTRAINT uk_users_email UNIQUE (email)
);

-- 2. Spring Session Tables
CREATE TABLE spring_session (
    primary_id CHAR(36) NOT NULL,
    session_id CHAR(36) NOT NULL,
    creation_time BIGINT NOT NULL,
    last_access_time BIGINT NOT NULL,
    max_inactive_interval INT NOT NULL,
    expiry_time BIGINT NOT NULL,
    principal_name VARCHAR(100),
    CONSTRAINT pk_spring_session PRIMARY KEY (primary_id)
);

CREATE UNIQUE INDEX idx_spring_session_session_id ON spring_session (session_id);
CREATE INDEX idx_spring_session_expiry_time ON spring_session (expiry_time);
CREATE INDEX idx_spring_session_principal_name ON spring_session (principal_name);

CREATE TABLE spring_session_attributes (
    session_primary_id CHAR(36) NOT NULL,
    attribute_name VARCHAR(200) NOT NULL,
    attribute_bytes BYTEA NOT NULL,
    CONSTRAINT pk_spring_session_attributes PRIMARY KEY (session_primary_id, attribute_name),
    CONSTRAINT fk_spring_session_attributes_spring_session FOREIGN KEY (session_primary_id) REFERENCES spring_session(primary_id) ON DELETE CASCADE
);
```

## 4. Backend Source Code Additions

### 4.1 Dependency Addition (server/pom.xml)
Include:
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-validation</artifactId>
</dependency>
```

### 4.2 Shared base entities and security helpers
1. `BaseEntity.java` in `com.krishnamurti.horizon.shared.domain` (holding `id`, `created_at`, `updated_at`).
2. `CurrentUserProvider.java` in `com.krishnamurti.horizon.shared.security`.
3. `SecurityConfig.java` in `com.krishnamurti.horizon.shared.security` (a permissive starter config).

## 5. Frontend Testing Configuration

### 5.1 Test Library Dependencies (client/package.json)
Install devDependencies:
- `vitest`
- `jsdom`
- `@testing-library/react`
- `@testing-library/dom`
- `@testing-library/jest-dom`
- `@vitest/coverage-v8`
- `@playwright/test`

### 5.2 Scripts (client/package.json)
Add:
- `"test": "vitest run"`
- `"test:watch": "vitest"`
- `"test:e2e": "playwright test"`

### 5.3 Config files
1. `client/vitest.config.ts` (merges `vite.config.ts` and configures environment as `jsdom`).
2. `client/src/setupTests.ts` (imports `@testing-library/jest-dom`).
3. `client/src/App.test.tsx` (renders `App` and asserts `"Get started"` heading is visible).
4. `client/playwright.config.ts` (e2e configuration mapping browser to `http://localhost:5173`).
5. `client/e2e/example.spec.ts` (navigates to `/` and asserts page heading).

## 7. Definition of Done
1. Maven wrapper can build backend, compile successfully, and run tests.
2. Flyway migrations compile and execute correctly on local PostgreSQL database (port 5433).
3. Frontend dependencies install successfully.
4. Vitest unit tests run and pass.
5. Playwright configuration is verified.
6. Documentation files are complete and readable in the workspace.
