# Flyway Database Schema Migrations

This skill defines the procedures and guidelines for introducing and validating database schema migrations within the Horizon backend.

## Conventions & Rules

1. **Schema Ownership:**
   - The database owns the schema. Hibernate schema generation is disabled (`spring.jpa.hibernate.ddl-auto=validate`).
   - Every schema change must be written as a Flyway SQL migration script.

2. **File Location & Naming:**
   - All migrations belong in `server/src/main/resources/db/migration/`.
   - Name format: `V<YYYYMMDDHHmmss>__<description>.sql` (e.g., `V20260620180000__create_task_table.sql`). Use UTC time for the timestamp.

3. **Identifier Strategy:**
   - Always use `BIGINT GENERATED BY DEFAULT AS IDENTITY` for entity primary keys (`Long` in Java).

4. **Timestamp Strategy:**
   - Always use `TIMESTAMPTZ` for dates and times to ensure UTC standardization.

5. **Naming Conventions:**
   - Tables: `snake_case`, singular (e.g., `checklist_item`).
   - Columns: `snake_case` (e.g., `completed_at`).
   - Primary Key Constraints: `pk_<table>` (e.g., `pk_task`).
   - Foreign Key Constraints: `fk_<table>_<referenced_table>` (e.g., `fk_checklist_item_task`).
   - Unique Constraints: `uk_<table>_<columns>` (e.g., `uk_tag_user_name`).
   - Indexes: `idx_<table>_<purpose>` (e.g., `idx_task_user_id`).

## Step-by-Step Procedure

1. **Create the Migration File:**
   - Generate a timestamp for the filename.
   - Create the `.sql` file under `server/src/main/resources/db/migration/`.
   - Write standard PostgreSQL statements. Keep all operations explicit.

2. **Verify JPA Entity Alignment:**
   - Ensure the JPA entity class annotations matching constraints and column names (e.g., `@Table(name = "...")`, `@Column(name = "...")`) reflect the migration precisely.

3. **Validate Build & Schema Compilation:**
   - Run the validation locally via Maven:
     ```powershell
     ./mvnw.cmd test-compile
     ```
     This triggers Hibernate's boot validation schema check against your entities.
